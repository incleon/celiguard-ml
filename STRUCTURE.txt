Directory Structure:
====================
├── 3 Service Directories (backend, frontend, train)
├── 1 Shared Directory (models)
├── 1 Git Directory (.git)
└── Root configuration and documentation files

Docker Services:
================
1. train       - One-time job to generate model
2. backend     - REST API on port 8000
3. frontend    - Web UI on port 8501

All services connected via 'celiguard-network' Docker network
Models shared via Docker volume mounted at ./models

Key Features:
=============
✓ Modular architecture with separated concerns
✓ Full Docker containerization
✓ One-command deployment (docker-compose up)
✓ Comprehensive documentation
✓ Health checks for all services
✓ Environment-based configuration
✓ Production-ready structure
✓ Scalable design

Quick Commands:
===============
Start all:      docker-compose up --build
Stop all:       docker-compose down
View logs:      docker-compose logs -f
Train only:     docker-compose up train
Backend only:   docker-compose up backend
Frontend only:  docker-compose up frontend

Using Makefile:
make build      Build all images
make up         Start all services
make down       Stop all services
make logs       View logs
make clean      Remove everything

Access Points:
==============
Frontend:       http://localhost:8501
Backend API:    http://localhost:8000
API Docs:       http://localhost:8000/docs
Health Check:   http://localhost:8000/health
